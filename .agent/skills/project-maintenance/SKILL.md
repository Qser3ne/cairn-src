# Project Maintenance

## 收尾流程

在完成任何项目修改后，执行以下检查：

1. 阅读 `Document/` 目录下与本次修改相关的文档，优先检查 `Document/功能分区/` 和 `Document/规范分区/`。
2. 判断本次修改是否改变了功能行为、约束规则、工作流、配置语义、接口契约、数据结构、测试期望或运行方式。
3. 如有变化，修正或补充 `Document/` 下对应文档，使文档与当前实现保持一致。
4. 检查本次修改是否可能影响项目内提示词，包括但不限于 `prompts/`、`prompt` 命名文件、Markdown 提示词模板、系统提示词、任务提示词、Worker 提示词或调度提示词。
5. 如提示词依赖的功能、字段、约束、输出格式或流程发生变化，修正相关项目内提示词。
6. 如果确认无需修改文档或提示词，在最终回复中简要说明已检查且无需更新。

## 输出要求

任务完成后的最终回复必须包含一行：

```text
√ <任务>
```

将 `<任务>` 替换为本次完成的具体任务名称或简短描述。
